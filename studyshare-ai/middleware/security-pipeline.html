Middleware Order

Recommended request pipeline

Execution order for robust auth, validation, and abuse protection in production.

Order Middleware Goal Applied on
1 Secure headers Set strict headers via Helmet-equivalent config All requests
2 Request ID / tracing Attach trace id for audit and diagnostics All requests
3 Rate limiter Throttle abusive traffic and burst attacks Auth, AI, upload routes
4 Body parser limits Cap payload size and reject malformed data All mutation routes
5 CSRF protection Protect browser-based form actions Session-auth POST/PATCH/DELETE
6 Authentication Verify JWT session and account state Protected routes
7 Role guard Restrict admin-only endpoints Admin routes
8 Input validation Schema validation and required fields All API routes
9 XSS sanitization Sanitize user-generated rich text Comments, descriptions, profile fields
10 Upload verifier MIME/type/size check and malware simulation flag Upload routes
11 Audit logging Record sensitive events and decisions Auth/admin/security events
12 Error boundary Standardized safe error response format Global
