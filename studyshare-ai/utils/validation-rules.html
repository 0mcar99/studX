Validation Spec

Input, content, and upload validation baseline

Rule definitions for form validation, sanitization, and anti-abuse controls.

Account validation

  • Email: RFC-compliant format + domain normalization
  • Password: 8 to 64 chars, upper/lower/number/symbol
  • Username: alphanumeric + underscore, 3 to 24 chars
  • Throttle account creation by IP and fingerprint

Note metadata validation

  • Title: 8 to 140 chars
  • Description: 30 to 2000 chars, HTML sanitized
  • Subject/topic/semester required and normalized
  • Tags: max 10, each max 32 chars

Upload validation

  • Allowlist MIME and extension alignment checks
  • File size limits by media type
  • Reject password-protected or unknown binary payloads
  • Mandatory malware simulation scan flag before publish

Comment and review validation

  • Content max length and profanity policy pass
  • XSS sanitize rich content before persistence
  • Rate limit comment posting bursts
  • Store moderation score for trust ranking
