Security Controls

Protection checklist and implementation rules

Security expectations for API routes, auth lifecycle, upload pipeline, and admin operations.

Password and session rules

  • Hash passwords with bcrypt and minimum cost factor 12
  • Never store raw credentials in logs
  • Rotate JWT signing keys using key versioning strategy
  • Set session expiry and revoke on logout/reset
  • Track failed login attempts and temporary lockouts

Request hardening

  • Use strict schema validation for every payload
  • Apply sanitization to all rich text fields
  • Enable SQL injection prevention through Prisma parameterization
  • Activate CSRF tokens for browser form mutations
  • Use rate limit policies for auth and AI endpoints

Upload security

  • Allowlist MIME types and extension combinations
  • Set strict file size limits by type
  • Store outside public web root and use signed URLs
  • Run malware simulation scanner before publish
  • Reject suspicious archives and executable payloads

Audit and risk monitoring

  • Record security-relevant events in SecurityLogs model
  • Capture ip, user agent, severity, and context metadata
  • Generate alerts for brute force and unusual geo access
  • Require admin action audit trail for moderation decisions
  • Review incident dashboard daily and close reports with reason
